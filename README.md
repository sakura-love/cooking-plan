## 这项目解决什么问题

很多菜谱 App 适合“收藏”，不适合“下厨”。

真正做饭时，常见问题不是找不到菜，而是：

- 菜谱步骤写得太粗，做一半就乱了
- 炒几分钟、焖多久、腌多长时间总要来回切 App
- 想提前安排这周吃什么，但菜谱、计划、买菜清单是分裂的
- 家里老人、小白、没配 API 的用户，根本用不上 AI 生成

`料理计划` 想做的是另一件事：

把一道菜拆成真正可以执行的任务流。

你不是在“看菜谱”，而是在“按步骤完成做饭”。

## 核心亮点

### 1. 任务式菜谱，不是静态图文

每道菜都可以拆成：

- 食材清单
- 分步骤操作
- 可打勾进度
- 带时长的计时节点

像“焯水 60 秒”“爆炒 90 秒”“腌制 10 分钟”“小火炖 40 分钟”这类步骤，可以直接在菜谱里执行，不用再跳去别的计时器。

### 2. 内置 200+ 菜谱，没接 AI 也能直接用

这个版本重点补强了离线体验。

- 内置 200+ 道家常菜、家庭可做特色菜、汤类、主食、凉菜、小吃、甜品
- 很多默认菜谱都补了更细的实操步骤
- 自动补齐缺失的内置菜谱，默认库更完整

对没有大模型 API 的用户，这一点尤其重要。

### 3. 首页支持分类目录，不搜索也能翻

菜谱多了以后，只有搜索是不够的。

现在首页支持：

- 分类目录快速切换
- 按分类分组浏览
- 搜索菜名、分类、简介、食材

不想搜索时，也能像翻实体菜单一样顺着找。

### 4. 做饭过程可继续

开始做一道菜以后，首页会显示“正在做”入口。

即使中途退出 App，再回来也能继续接着做，不需要重新翻菜谱。

### 5. 菜谱、三餐计划、买菜清单打通

你可以直接：

1. 收藏常做菜
2. 把菜加入这周的早餐 / 午餐 / 晚餐
3. 自动汇总当天买菜清单
4. 买菜时逐项勾选

这个闭环比单纯“存菜谱”实用得多。

## 安装

### 直接安装 APK

发布包位于 [releases](./releases/) 目录。

当前最新版本：

- `releases/料理计划-v1.2.8.apk`

如果手机里已经装过正式版，请优先继续安装 `release` APK 覆盖升级，不要用 `debug` 包直接覆盖。

### 本地构建

```bash
./gradlew assembleDebug
```

正式签名构建：

```bash
./gradlew assembleRelease
```

## 项目结构

```text
app/src/main/java/com/cooking/plan/
├─ data/
│  ├─ ai/          # AI 菜谱生成
│  ├─ local/       # Room 数据、默认菜谱
│  ├─ settings/    # 本地设置
│  └─ timer/       # 烹饪计时与前台服务
├─ di/             # 依赖注入
└─ ui/
   ├─ home/        # 首页与目录浏览
   ├─ plan/        # 每周计划与单餐详情
   ├─ recipe/      # 菜谱编辑 / 详情
   ├─ shopping/    # 采购清单
   ├─ favorites/   # 收藏
   ├─ guide/       # 新手指引
   └─ settings/    # 设置
```
