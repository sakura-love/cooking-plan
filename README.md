## 它解决什么问题

很多菜谱 App 更适合“收藏”，不适合“真正下厨”。

真正做饭时，常见问题通常是：

- 菜谱步骤太粗，做到一半就乱了
- 蒸多久、焖多久、炒多久总要来回切 App
- 想提前安排一周吃什么，但菜谱、计划、买菜清单是分开的
- 想要离线、本地、无广告的做饭工具，却很难找到
- 没配大模型 API 时，AI 菜谱功能基本用不上

`料理计划` 做的不是“再做一个菜谱收藏夹”，而是：

把一道菜拆成可以执行的做饭任务。

你不是在看菜谱，而是在按步骤把饭做出来。

## 核心亮点

### 任务式菜谱

每道菜都可以拆成：

- 食材清单
- 分步骤操作
- 勾选进度
- 需要计时的节点

像“焯水 60 秒”“爆炒 90 秒”“腌制 10 分钟”“小火炖 40 分钟”这类步骤，可以直接在做饭过程中执行，不用反复切换别的工具。

### 操作计时器

需要固定时长的步骤可以直接启动计时器，比如焯水、蒸、煮、焖、炖、腌制等。

做饭时不用再手动记时间，也不用来回切到系统时钟。

### 每周计划

可以按周安排早餐、午餐、晚餐。

同一餐支持添加多道菜，方便提前把这一周吃什么定下来。

### 自动买菜清单

安排好某一天或一周的菜谱后，可以直接生成对应的买菜清单。

食材会自动汇总，买菜时逐项打勾，减少漏买和重复记忆。

### 收藏常做菜

常做的菜可以收藏，方便下次快速找到，不用每次重新搜索。

### 200+ 本地菜谱

内置 200+ 道本地菜谱，包含：

- 家常菜
- 主食
- 汤
- 甜点
- 凉菜
- 小吃

除 AI 生成菜谱外，所有核心功能都可以离线使用。

### 可选 AI 生成菜谱

如果本地没有想做的菜，可以在设置里配置大模型 API，生成新的菜谱内容。

这个功能是可选的，不配置也不影响离线使用。

### 本地优先

项目强调本地化运行：

- 不需要登录
- 没有广告
- 菜谱、收藏、计划都保存在本地
- 除可选 AI 生成功能外，默认不依赖联网

## 安装

### 直接安装 APK

发布包位于 [releases](./releases/) 目录。

当前最新版本：

- `releases/料理计划-v1.2.8.apk`

如果手机里已经安装过正式版，请继续安装 `release` APK 覆盖升级，不要用 `debug` 包直接覆盖。

### 本地构建

```bash
./gradlew assembleDebug
```

正式签名构建：

```bash
./gradlew assembleRelease
```

## 项目结构

```text
app/src/main/java/com/cooking/plan/
├─ data/
│  ├─ ai/          # AI 菜谱生成
│  ├─ local/       # Room 数据、默认菜谱
│  ├─ settings/    # 本地设置
│  └─ timer/       # 烹饪计时与前台服务
├─ di/             # 依赖注入
└─ ui/
   ├─ home/        # 首页与目录浏览
   ├─ plan/        # 每周计划与单餐详情
   ├─ recipe/      # 菜谱编辑 / 详情
   ├─ shopping/    # 采购清单
   ├─ favorites/   # 收藏
   ├─ guide/       # 新手指引
   └─ settings/    # 设置
```
